# Code Principles

이 프로젝트는 다음의 핵심 개발 원칙을 준수하여 작성됩니다.

## 1. 명확한 변수명 (Naming)
**목적물과 행위가 확실하게 드러나는 변수명을 사용합니다.**
- 변수명만으로 해당 변수의 역할과 생명주기를 파악할 수 있어야 합니다.
- 모호한 축약어나 일반적인 명사(예: `data`, `item`) 사용을 지양합니다.
- **Good:** `userProfileImage`, `fetchTrackList`, `isRegionSelected`
- **Bad:** `img`, `list`, `flag`

## 2. 관심사 분리 (Separation of Concerns)
**각 모듈은 단 하나의 역할에 집중합니다.**
- 비즈니스 로직, 데이터 처리, UI 렌더링을 명확히 분리합니다.
- 하나의 컴포넌트나 함수가 너무 많은 책임을 지지 않도록 지속적으로 리팩토링합니다.
- 스타일, 로직, 뷰를 분리하여 유지보수성을 높입니다.

## 3. 의존성 관리 (Depdendency Management)
**직접 의존을 피하고 추상화와 의존성 주입을 지향합니다.**
- 상위 모듈이 하위 모듈의 구체적인 구현에 의존하지 않도록 합니다.
- 결합도를 낮추기 위해 인터페이스나 훅(Hook)을 통한 주입 패턴을 활용합니다.
- 이를 통해 유연하고 테스트 가능한 코드를 유지합니다.

## 4. 주석 없는 코드 (Self-Documenting Code)
**변수명과 함수명이 주석을 대신해야 합니다.**
- "이 코드는 ~을 합니다"라는 주석 대신, 그 내용을 담은 함수명으로 리팩토링합니다.
- 복잡한 조건식은 의미를 알 수 있는 변수로 추출하여 가독성을 높입니다.
- 주석은 '무엇(What)'이 아닌 '왜(Why)'를 설명할 때만 제한적으로 사용합니다.
